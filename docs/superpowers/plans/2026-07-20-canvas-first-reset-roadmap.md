# Canvas-First Reset Delivery Roadmap

**Source specification:** `docs/superpowers/specs/2026-07-17-canvas-first-product-reset-design.md`

The reset is divided into four independently testable phases. Each phase must leave the default application route usable and preserve the immutable source artwork model established in phase one.

## Phase 1: Editor Foundation

Deliver a local canvas editor with raster import, a versioned project and asset model, non-destructive crop/transform/basic adjustments, named variations, undo/redo, IndexedDB autosave, local project reopening, and responsive desktop/mobile controls.

Exit criteria:

- The default route opens the editor instead of a landing page or workflow mode picker.
- A PNG, JPEG, or WebP can be imported, edited, duplicated as a variation, autosaved, reloaded, and reopened without changing the stored source blob.
- The old production workbench has no active UI route, while static informational routes continue to work.
- Typecheck, build, unit tests, and the phase-one Playwright flow pass.

Detailed plan: `docs/superpowers/plans/2026-07-20-canvas-first-editor-foundation.md`

## Phase 2: Creative Layers And Comparison

Add ordered image and text layers, deterministic Looks, seeded grain and distress, adjustable vector trace layers, SVG master export, and Compare Board views for two to four variations.

Exit criteria:

- Layer order, visibility, transforms, and seeded effects survive save/reopen and match final rendering.
- Text and trace layers remain editable and SVG export is structurally valid.
- Compare Board normalizes framing across variation aspect ratios on desktop and mobile.

## Phase 3: Products, Mockups, And Validated Export

Add independent product variants, Printify-oriented and custom targets, placement controls, restored photographic shirt blanks, representative mockups, worker-backed high-resolution rendering, Pica fixture gating, PNG/SVG export, and generated-file validation receipts.

Exit criteria:

- One design variation can drive independent shirt, hoodie, mug, poster, and custom placements.
- A product preview never mutates the design variation or another product placement.
- Exported dimensions, alpha behavior, DPI metadata, file size, and local resampling metadata are validated from generated bytes.

## Phase 4: Print Lens, Treatments, And Retirement

Add physical-size Print Lens overlays, ink-aware preview, DTG garment treatments, treatment reset and comparison, then remove unreachable production-job, approval, package, profile, batch, and Gemini code and dependencies.

Exit criteria:

- Print findings are deterministic, localized, labeled as heuristics, and never mutate artwork without an explicit fix action.
- Product treatments remain independent and reversible while the creative master stays unchanged.
- The shipped bundle and repository contain no active production-workbench or AI feature surface.
- Full desktop/mobile acceptance, accessibility checks, fixture comparisons, and generated-file validation pass.
