# Phase 3B Validated T-Shirt PNG Export Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Generate, validate, and download the active T-shirt placement as one of three fixed transparent PNG files without blocking the editor or including the photographic shirt.

**Architecture:** Keep the existing SVG export path unchanged outside Product mode. Product mode captures an immutable export snapshot, sends it to a dedicated module worker, renders authoritative assets and the saved Look at final resolution, inserts PNG resolution metadata, and returns bytes that the main thread parses before enabling download. Focused model, PNG, coordinator, renderer, worker, and dialog modules keep final rendering out of `EditorApp`.

**Tech Stack:** TypeScript 5.8, React 19, Canvas 2D and OffscreenCanvas, Pica 10.0.2, module workers, existing compositor and Look recipes, Node test runner with `tsx`, Playwright Chromium, Vite 8.

## Global Constraints

- Execution base is commit `13af020`.
- The implementation contains exactly seven reviewed tasks.
- The only outputs are `4500x5400 @ 300 DPI`, `3000x3600 @ 300 DPI`, and `1500x1800 @ 150 DPI`.
- The first two presets are Production; `1500x1800` is always labelled `Proof only`.
- Every file is an eight-bit RGBA PNG with meaningful transparency and at most `100 MiB`.
- Product placement maps with `baseSide = min(width, height)`, `renderedSide = baseSide * scale`, `centerX = width * x`, and `centerY = height * y`.
- Shirt images, shirt color, shadows, texture, and preview calibration never enter the export snapshot, fingerprint, renderer, or downloaded file.
- Final generation uses a dedicated worker with a `90_000 ms` timeout and no main-thread or legacy-production fallback.
- Raster resizing uses exact dependency `pica@10.0.2`, `lanczos3`, `1024`-pixel tiles, and no nested Pica worker pool.
- The generated file contains exactly one meter-based `pHYs` chunk: `11811` pixels per meter for 300 DPI or `5906` for 150 DPI.
- The actual returned PNG bytes are parsed before download; the parsed Blob is the downloaded Blob and is never re-encoded.
- Relevant variation, asset, Look, preset, or placement changes invalidate the result; shirt-color-only changes do not.
- Generated bytes, Blobs, object URLs, canvases, and decoded images are session resources and are never persisted.
- No custom dimensions, additional product, mockup download, Print Lens, Print Treatment, provider API, cloud, AI, or unrelated legacy retirement enters this phase.
- New Phase 3B editor modules cannot import production jobs, profiles, proofs, packages, batches, provider APIs, Gemini, AI modules, `services/mockups.ts`, or legacy processing services.
- Every task ends with focused tests, `npm run typecheck`, `git diff --check`, review, and one scoped commit.

  center: { x: number; y: number };
  renderedSide: number;

export const resolveTShirtExportGeometry = (
  preset: TShirtExportPreset,
  placementValue: ProductPlacement,
): TShirtExportGeometry => {
  const placement = normalizeProductPlacement(placementValue);

      y: preset.height * placement.y,

    renderedSide: baseSide * placement.scale,

- [ ] **Step 4: Implement authoritative `pHYs` replacement**

`writePngResolution` must:

1. Parse and validate the source structure.
2. Copy the PNG signature and IHDR.
3. Drop every existing `pHYs` chunk.
4. Insert one 9-byte `pHYs` chunk immediately after IHDR.
5. Write equal X/Y pixels per meter, unit byte `1`, and a valid CRC.
6. Copy every other chunk through terminal IEND.
7. Return a fresh `Uint8Array` without mutating caller bytes.

Use this chunk payload:

```ts
const data = new Uint8Array(9);
const view = new DataView(data.buffer);
view.setUint32(0, pixelsPerMeter, false);
view.setUint32(4, pixelsPerMeter, false);
data[8] = 1;
```

export type TShirtExportOutcome =
  | {
      status: 'ready';
      fingerprint: string;
      pngBytes: Uint8Array;
      metadata: TShirtExportRenderMetadata;
    }
  | { status: 'failed'; fingerprint: string; stage: TShirtExportStage; message: string }

export class TShirtExportCoordinator {

`PNG generation failed while <stage label>.`.
